# Internationalisation (FR / EN)

## Stack

- `i18next` + `react-i18next` + `i18next-browser-languagedetector`
- Fichiers : `src/locales/fr.json` et `src/locales/en.json` (namespace unique `translation`)
- Initialisation : `src/i18n/index.ts` (importé en premier dans `src/main.tsx`)
- Langue persistée : clé localStorage `auclaire-lang`

## Utilisation

```tsx
import { useTranslation } from 'react-i18next';

const { t, i18n } = useTranslation();
return <span>{t('nav.items.dashboard')}</span>;
```

Interpolation : `t('dailyReport.invoicesCount', { count: 3 })`

## Compléter l’app

1. Ajouter la clé dans **fr.json** et **en.json** (même structure).
2. Remplacer le texte en dur par `t('...')`.
3. Préférer des clés groupées : `dashboard.widgets.*`, `nav.items.*`, etc.

Les écrans non encore migrés restent dans leur langue d’origine jusqu’à ajout des clés.

### État de la couverture (à jour approx.)

**Déjà branchés sur i18n (UI principale)** : `App`, `CRMLayout`, `LanguageSwitcher`, `Login`, `PendingApproval`, `Dashboard`, `Settings`, `ResourcesHub`, **`SalesProcess`**, **`FlashCalculator`**, **`ClientQuote`**, **`ProductCatalog`**, **`ProjectDetails`** (partiel — voir `projectDetailsPage.*`), **`Studio`** (`studioPage.*`), **`Formation`** (partiel — `formationPage.*`), `ManufacturerDashboard`, `DailyReportSheet`, `ProjectsList`, `MessageCenter`, `ClientsList`, `CreateClient`, `CreateProject`, `ClientForm`, `Leads`, `DebugPage`, `BetaFeedback`, `StatusBadge`, `GlobalSearch`, `ProjectChat`, `ErrorBoundary`, `OfflineIndicator`, **`CreateInvoice`**, **`InvoicesList`**, **`ExpensesList`**, **`SuppliersList`** (clés `createInvoicePage.*`, `invoicesPage.*`, `expensesPage.*`, `suppliersPage.*`), **CRM** : `LeadsDashboard`, `LeadDetails`, `Dialer`, `Tasks` (clés `crmLeadsDashboard.*`, `crmLeadDetails.*`, `dialer.*`, `tasksPage.*`, `leadStatus.*`), **clients / portail / affiliés** : `ClientDetails`, `ClientPortal`, `AffiliatesList`, `AffiliateDetails`, `AffiliateDashboard`, `RegisterAffiliate`, **analytics / production** : `CashFlowForecast`, `ProductionCalendar`, **`AnalyticsDashboard`** (y compris insights IA via `analyticsPage.insights.*` / `generateInsights()`).

**Reste typique pour viser 100 % bilingue** (hors données métier / historiques) :
- **`ProjectDetails`** — la page principale est couverte par `projectDetailsPage.*` ; **`DesignApprovalPanel`** utilise `projectDetailsPage.designApproval.*` (toasts, notifications, UI)
- **E-mails & notifications** (Edge `send-email`, sujets/corps) — clés FR/EN côté serveur ou templates par langue
- **`ActivityLog`** — le champ `details` peut rester dans la langue du log (choix produit)
- **Contenu admin / PDF / exports** — selon périmètre souhaité
- **Revues ponctuelles** — `grep` de chaînes FR en dur dans `src/` (hors tests / commentaires)

**`Formation`** — onglet Guides : **fait** (`formationPage.guides.*`).

L’app ne sera **100 % bilingue** que lorsque chaque écran / composant affichant du texte utilisateur aura été passé sur `t()` + clés FR/EN (ou données localisées).
